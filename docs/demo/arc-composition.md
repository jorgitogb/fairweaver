# ARC Composition Overview

## Entity Model

An ARC (Annotated Research Context) RO-Crate is a structured research data package. It follows the [DataPLANT ARC Specification v3.0](https://github.com/nfdi4plants/ARC-specification) and consists of:

```
ARC Root
├── .arc/                         # ARC metadata
├── isa.investigation.xlsx        # Investigation-level metadata
├── ro-crate-metadata.json        # RO-Crate JSON-LD
├── studies/                      # Study data
│   └── <study_id>/
│       ├── isa.study.xlsx
│       └── resources/
├── assays/                       # Measurement data
│   └── <assay_id>/
│       ├── isa.assay.xlsx
│       └── dataset/
├── workflows/                    # Computational workflows
│   └── <workflow_id>/
│       └── workflow.cwl
└── runs/                         # Workflow executions
    └── <run_id>/
        └── run.cwl
```

## JSON-LD Entity Types

In `ro-crate-metadata.json`, entities are organized as:

| Entity | @type | additionalType | Purpose |
|--------|-------|----------------|---------|
| Root Dataset | Dataset | — | Top-level container with citation metadata |
| Investigation | Dataset | Investigation | Project-level administrative metadata |
| Study | Dataset | Study | Experimental design and protocols |
| Assay | Dataset | Assay | Measurement technique and data files |
| Person | Person | — | Researchers, authors, contacts |
| Organization | Organization | — | Affiliations, publishers |
| File | File | — | Data files (raw, processed, results) |
| ScholarlyArticle | ScholarlyArticle | — | Publications, references |

## Default ARC vs FAIRagro ARC Comparison

| Aspect | Default ARC (DataPLANT) | FAIRagro ARC (Extended) |
|--------|------------------------|-------------------------|
| **Investigation fields** | name, description, identifier | + creator, license, datePublished, contacts, publications |
| **Study fields** | name, description | + designDescriptors, crop info, location |
| **Assay fields** | name | + measurementTechnique, measurementMethod, sensor metadata |
| **Crop metadata** | Not required | Required for agronomy domain |
| **Sensor metadata** | Not required | Required for phenotyping |
| **Validation rules** | Directory structure | + Field-level rules, confidence scoring |
| **Publishability** | Not enforced | Explicit checks for publication readiness |
| **Reproducibility** | Not enforced | CWL workflow & run verification |
